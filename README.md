## 1. Chosen Dataset

For this project, we use the **YAMBDA (Yandex Music Big Data Analytics)** dataset ‚Äî  
a large-scale, real-world dataset of user interactions from the **Yandex.Music** streaming platform.

---

## 2. Why This Dataset

The **YAMBDA** dataset was selected because it combines **realism, scale, and accessibility**, making it perfectly suited for research in recommendation systems and user modeling.

**Key reasons:**
- üìä **Real user behavior:** based on genuine listening and ‚Äúlike‚Äù events from a real platform, not synthetic data.  
- üíæ **Large scale:** over **50 million** interactions, enabling testing of algorithms under realistic data loads.  
- üïí **Temporal data:** includes timestamps for each interaction, allowing time-based and session-aware modeling.  
- ‚öôÔ∏è **Versatility:** suitable for both classical (ALS, collaborative filtering) and modern (deep learning) recommendation methods.  
- üåç **Open and reproducible:** available via Hugging Face for transparent, shareable experiments.

In short ‚Äî YAMBDA offers both **data realism** and **industrial-scale complexity**, essential for evaluating recommender algorithms in practice.

---

## 3. Dataset Structure

The dataset is stored in **Parquet** format under `flat/50m/likes.parquet` and contains the following core fields:

| Column | Type | Description |
|---------|------|-------------|
| `user_id` | `int64` | Unique anonymized user identifier |
| `item_id` | `int64` | Unique track identifier |
| `timestamp` | `int64` | UNIX timestamp of the event |
| `event_type` | `string` | Type of interaction (e.g., `like`, `listen`, `skip`) |

Some variations may include additional metadata fields depending on the subset.
